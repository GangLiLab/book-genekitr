3 Gene ID conversion

The popular gene ID types include Symbol, NCBI Entrez and Ensembl. ID conversion among different types is very common.

3.1 Current tools

Now there are many tools could handle id conversion including biomaRt, clusterProfiler. .

biomaRt is frequently used to access Ensembl database while clusterProfiler depends on NCBI database via Bioconductor annotation packages.

Genekitr combines both Ensembl and NCBI database to provide more comprehensive gene id mapping. Also id conversion supports many species (see also session 1.2)

3.2 Basic usage

transId is a sub function of genInfo, which has five arguments:

  • id: gene id (symbol, Entrez or Ensembl) or protein id

  • transTo: the conversion target type. User could select one or more from ‚Äúsymbol‚Äù, ‚Äúentrez‚Äù, ‚Äúensembl‚Äù or ‚Äúuniprot‚Äù

  • org: organism name, default is human

  • unique: TRUE or FALSE. If TRUE, only returns one matched id. Default is FALSE.

  • keepNA: TRUE or FALSE. Some id has none matched id, save it or not. Default is FALSE.

id <- c('AKT3','SSX6P','FAKE_ID','BCC7')

# Here "ens" represents "ensembl", if you do not want to type too many characters like me, just spell first several ones
transId(id,'ens')
##   input_id         ensembl
## 1     AKT3 ENSG00000275199
## 2     AKT3 ENSG00000117020
## 3    SSX6P ENSG00000171483
## 4     BCC7 ENSG00000141510
# Due to NA was removed, the result order is slightly different from input id. If you want to get the same order, you can save NA:
transId(id,'ens',keepNA = TRUE)
##   input_id         ensembl
## 1     AKT3 ENSG00000275199
## 2     AKT3 ENSG00000117020
## 3    SSX6P ENSG00000171483
## 4  FAKE_ID            <NA>
## 5     BCC7 ENSG00000141510
# If you want to get one-to-one mapping and keep input id order:
transId(id,'ens',keepNA = TRUE,unique = TRUE)
##   input_id         ensembl
## 1     AKT3 ENSG00000275199
## 2    SSX6P ENSG00000171483
## 3  FAKE_ID            <NA>
## 4     BCC7 ENSG00000141510

3.3 Features

3.3.1 Feature1: convert to several types

Just pass the wanted types to transTo parameter:

transId(id, transTo = c('ens','ent'))
##   input_id         ensembl entrezid
## 1     AKT3 ENSG00000275199    10000
## 2     AKT3 ENSG00000117020    10000
## 3    SSX6P ENSG00000171483   280657
## 4     BCC7 ENSG00000141510     7157

3.3.2 Feature2: distinguish from symbol and alias

Gene alias is an alternative name for official gene symbol, like ‚ÄúBCC7‚Äù is actually alias for ‚ÄúTP53‚Äù. Sometimes gene alias is much famous than official symbol, for example, ‚ÄúPDL1‚Äù is the alias for ‚ÄúCD274‚Äù in fact.

Here, transId could utilize both NCBI and Ensembl alias metadata to easily recognize them, user just need to set transTo = 'symbol'

transId(c("BCC7","TP53","PD1","PDL1","TET2"),'sym')
##   input_id symbol
## 1     BCC7   TP53
## 2     TP53   TP53
## 3      PD1  PDCD1
## 4      PD1   SNCA
## 5      PD1 SPATA2
## 6     PDL1  CD274
## 7     TET2   TET2

3.3.3 Feature3: no worry for Ensembl version

An Ensembl stable ID consists of five parts: ENS(species)(object type)(identifier). (version) For example, ENSG00000141510.11 is TP53 while the version .11 may confuse many tools. However using transId, user will not worry this issue any more.

transId('ENSG00000141510.11','symbol')
##          input_id symbol
## 1 ENSG00000141510   TP53

3.4 Real Example

3.4.1 Example data

We got a differential expression analysis result from GSE42872.

data(deg,package = 'genekitr')
id = deg$ENTREZID
length(id)
## [1] 18591

Here we will take gene symbols in the result to compare id conversion efficiency of various tools.

3.4.2 bitr from clusterProfiler 

### Using bitr
suppressWarnings(library(org.Hs.eg.db))
suppressMessages(library(clusterProfiler))
bitr_res = bitr(id,fromType = 'ENTREZID',toType = 'SYMBOL',OrgDb = 'org.Hs.eg.db')
bitr_sym = unique(bitr_res$SYMBOL)
length(bitr_sym)
## [1] 18579

3.4.3 transId from genekitr 

transid_res = transId(id,'sym')
transid_sym = unique(transid_res$symbol)
length(transid_sym)
## [1] 18600

3.4.4 compare results 

plotVenn(list(bitr_sym=bitr_sym,transid_sym=transid_sym))

There are 114 genes only in bitr result:

bitr_sym[!bitr_sym%in%transid_sym] %>% head()
## [1] "FAM122C"      "LOC100287896" "TMEM159"      "C8orf31"      "RRNAD1"       "CCDC58"
# let's take a look for the first one FAM122C, which is converted from gene 159091
bitr_res %>% filter(SYMBOL%in%'FAM122C') %>% pull(ENTREZID)
## [1] "159091"

When we search NCBI 159091 and we could see the official symbol is ‚ÄúPABIR3‚Äù now while ‚ÄúFAM122C‚Äù is an alias. When we look back into transId result, it keeps update with NCBI web.

transid_res %>% filter(input_id%in%'159091') %>% pull(symbol)
## [1] "PABIR3"

Besides, there are 140 unique symbols in transId result:

transid_sym[!transid_sym%in%bitr_sym] %>% head()
## [1] "PABIR3"    "LIPT2-AS1" "LDAF1"     "LINC02904" "METTL25B"  "MIX23"
# take a look at the second one: LIPT2-AS1
transid_res %>% filter(symbol == "LIPT2-AS1") %>% pull(input_id)
## [1] "100287896"

NCBI web also showed the same result with transId: 100287896, while bitr gets LOC100287896 symbol for it.

We have total 18591 entrez id as input, while we get 18578 ids in transId result, why?

transid_id <- unique(transid_res$input_id)
length(transid_id)
## [1] 18578
id[!id%in%transid_id]
##  [1] "23285"     "283174"    "105377021" "102724237" "101928841" "643763"    "54748"     "338809"   
##  [9] "387638"    "246734"    "84953"     "105374089" "10896"

When we search on NCBI web, we found the reason, they are replaced with new one. For example, the first one 23285 is discontinued on 9-Nov-2019 and replaced with 284697.

# As we get the latest id, we could convert it immediately!
transId('284697','sym')
##   input_id symbol
## 1   284697  BTBD8
